Detailed Description

template<class... Options>
struct awesm::machine_conf_tpl< Options >

The configuration type template for an machine.

State machine definitions must expose a conf type alias to an instance of this template. To instantiate this template, you have to write a chain of subtypes, starting from machine_conf (a type alias of machine_conf_tpl<>), where each subtype sets/activates an option.

Example:

using conf = awesm::machine_conf
::transition_tables<transition_table_t> //sets the transition_tables option
::context<context> //sets the context option
::pretty_name //activates the pretty_name option
;

Disables run-to-completion.

This makes the state machine much faster, but you have to make sure you never call machine::process_event() recursively.

Use it at your own risk!

Requires the machine to call a user-provided on_event() member function whenever it is about to process an event. Run-to-completion guarantee applies.

Template Parameters
EventFiltersthe list of events for which we want the machine to call on_event()

One of these expressions must be valid, for every given event type:

machine_def.on_event(fsm, event);
machine_def.on_event(event);
machine_def.on_event();

This hook can be useful when there are certain event types that you always want to process the same way, whatever the active state(s) of the state machine.

Example:

struct machine_def
{
using conf = machine_conf
::on_event<event_type_0, event_type_1>
//...
;
void on_event(const event_type_0& event)
{
//...
}
template<class Sm>
void on_event(Sm& fsm, const event_type_1& event)
{
//...
}
//...
};

If manually listing all the event type you're insterested in is too inconvenient, you can use on_event_auto.

Requires the machine to call a user-provided on_exception() member function whenever it catches an exception.

The following expression must be valid:

machine_def.on_exception(std::current_exception());

If this option isn't set, the machine will send itself a events::exception event, like so:

process_event(events::exception{std::current_exception()});
