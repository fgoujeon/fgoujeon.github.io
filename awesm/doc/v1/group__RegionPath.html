Detailed Description

A path to a region or subregion of an machine.

In some places in the API, you must provide or are provided a path to a region.

Conceptually, a path to a region consists in a sequence of (machine_def, region_index) pairs, starting from a region of the root machine. This is the same concept as a path to a directory on a filesystem, which consists in a sequence of directories, starting from the root directory.

In terms of C++ code, a (machine_def, region_index) pair is defined with an instance of the region_path_element struct template. These instances are sequenced in the template argument list given to the region_path_tpl template instance. Here is an example:

//A submachine with multiple regions, used machine_def
struct some_submachine
{
//...
};
//An machine with a single region
struct machine_def
{
//...
};
using machine_t = awesm::machine<machine_def>;
//Path to the second region (index 1) of some_submachine
using region_path_t = awesm::region_path_tpl
<
awesm::region_path_element<machine_def, 0>,
awesm::region_path_element<some_submachine, 1>
>;
