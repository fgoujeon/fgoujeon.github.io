nodiscardconstexpr

Specifies an action to be called whenever an exception is caught by machine.

As the suffix of the function suggests, the action must take references to:

  • the machine;
  • the exception pointer.

Example:

.exception_hook_mx([](auto& mach, const std::exception_ptr& eptr)
{
//...
})

If this action isn't set, maki::machine will send itself a maki::events::exception event, like so:

process_event(maki::events::exception{std::current_exception()});
maki::events::exception
Event sent by the default exception handler of machine whenever it catches an exception.

Adds a hook to be called whenever maki::machine is done processing an event of type Event.

Hook must have the following form:

template<class Machine, class Event>
void hook(Machine& mach, const Event& event, bool processed);

The processed parameter indicates whether a state transition or any kind of action invocation (note: a hook isn't an action) occurred during the processing of the event.

Users typically add a hook for every event of interest.

Example:

constexpr auto conf = maki::machine_conf{}
//...
.post_processing_hook_mep<some_event_type>([](auto& mach, const some_event_type& event, const bool processed)
{
//...
})
.post_processing_hook_mep<some_other_event_type>([](auto& mach, const some_other_event_type& event, const bool processed)
{
//...
})
;

Specifies a hook to be called after any external state transition.

Hook must have the following form:

template
<
class RegionImpl,
class SourceStateImpl,
class Event,
class TargetStateImpl
>
void hook
(
context& ctx,
const maki::region<RegionImpl>& region,
const maki::state<SourceStateImpl>& source_state,
const Event& event,
const maki::state<TargetStateImpl>& target_state
);

This hook can be useful for logging state transitions, for example:

.post_state_transition_hook_crset
(
[](context& /*ctx*/, const auto& region, const auto& source_state, const auto& /*event*/, const auto& target_state)
{
std::cout
<< "End of transition in FSM/"
<< region.path().to_string()
<< ": "
<< source_state.pretty_name()
<< " -> "
<< target_state.pretty_name()
<< "\n";
}
)

nodiscardconstexpr

Specifies whether run-to-completion is enabled.

Disabling run-to-completion makes the state machine much faster, but you have to make sure you never call maki::machine::process_event() recursively.

Disable it at your own risk!
