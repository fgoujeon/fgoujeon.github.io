Specifies an action to be called whenever an exception is caught by machine.

As the suffix of the function suggests, the action must take references to:

  • the machine;
  • the event.

Example:

.exception_action_me([](auto& mach, const auto& event)
{
//...
})

Adds an action to be invoked whenever a call to maki::machine::process_event() doesn't lead to any state transition or call to any kind of action.

Users typically add an action for every event of interest.

Example:

constexpr auto conf = maki::machine_conf{}
//...
.fallback_transition_action_me<some_event_type>([](auto& mach, const some_event_type& event)
{
//...
})
.fallback_transition_action_me<some_other_event_type>([](auto& mach, const some_other_event_type& event)
{
//...
})
;

◆ run_to_completion()

template<IMPLEMENTATION_DETAIL >
constexpr auto maki::machine_conf< IMPLEMENTATION_DETAIL >::run_to_completion ( const bool value) const
constexpr

Specifies whether run-to-completion is enabled.

Disabling run-to-completion makes the state machine much faster, but you have to make sure you never call maki::machine::process_event() recursively.

Disable it at your own risk!
